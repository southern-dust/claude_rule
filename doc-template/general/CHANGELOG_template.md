## 版本说明

### 版本号规则

本项目遵循语义化版本 (Semantic Versioning)：`MAJOR.MINOR.PATCH`

- **MAJOR**: 不兼容的API变更
- **MINOR**: 向后兼容的新功能
- **PATCH**: 向后兼容的Bug修复

### 变更类型说明

#### 新增 (Added)
- 新功能
- 新特性
- 新增的API或接口

#### 变更 (Changed)
- 现有功能的变更
- 行为的修改
- 默认值的调整

#### 废弃 (Deprecated)
- 即将在未来版本移除的功能
- 推荐使用替代方案

#### 移除 (Removed)
- 已删除的功能
- 不再支持的特性

#### 修复 (Fixed)
- Bug修复
- 错误处理改进
- 稳定性提升

#### 安全 (Security)
- 安全漏洞修复
- 安全性增强
- 权限相关的改进

### 变更记录格式

每个变更记录应包含：

```markdown
- [变更类型] 简短描述 (#PR编号)

  详细说明（可选）
  - 子项1
  - 子项2
```

示例：

```markdown
### 新增
- 新增用户认证功能 (#123)
  - 支持OAuth2登录
  - 支持JWT令牌
- 新增数据导出功能 (#124)

### 修复
- 修复内存泄漏问题 (#125)
- 修复并发访问的竞态条件 (#126)
```

### 发布流程

1. 更新本文件，添加新版本的变更记录
2. 清空"未发布"部分
3. 更新版本号
4. 创建Git标签：`git tag v1.0.0`
5. 推送标签：`git push origin v1.0.0`
6. 发布GitHub Release

### 链接格式

可以添加链接到相关的问题或PR：

```markdown
- 新增功能 (#[PR编号])
- 修复问题 (#[Issue编号])
```
